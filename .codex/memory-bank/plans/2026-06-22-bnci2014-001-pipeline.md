# BNCI2014_001 Motor Imagery Pipeline Run

Status: in_progress
Last updated: 2026-06-22
Next stage: 7 - Final Notebook, Report, And Memory Update (awaiting review)

## Goal

Run a leakage-aware benchmark on BNCI2014_001 through MOABB and evaluate how far the repository's
EEG pipeline can transfer to a standard motor-imagery dataset: loading, task adapter, features or
spectra, subject-aware evaluation, reproducible artifacts, and notebook reporting.

## Scope

Included:

- MOABB loading for BNCI2014_001.
- Four-class motor-imagery classification.
- Primary protocol: subject-wise generalization with no subject overlap between train and test.
- Secondary protocol: session-aware within-subject or cross-session evaluation if Stage 2 confirms
  a clean contract.
- Minimal baseline: CSP plus LDA or Logistic Regression.
- Project-style feature or spectral run: FFT/features first, then one lightweight spectral neural
  pilot only after the classical baseline works.
- Artifacts under `artifacts/experiments/bnci2014_001/`.
- Executed notebooks under `notebooks/`.

Excluded:

- Reuse of random-imagery 6x6 target logic.
- State-of-the-art hyperparameter search or claims.
- Strong state-of-the-art claims.

## Acceptance Criteria

- Dataset adapter emits deterministic rows with subject, session, run, epoch, label, sampling
  frequency, and channel metadata.
- Splits are audited for subject/session leakage.
- At least one classical baseline produces reproducible fold-level metrics.
- At least one project feature or spectral representation is run end to end.
- Notebook output is executed top-to-bottom and includes counts, splits, metrics, confusion matrix,
  and limitations.
- `uv run ruff check .` and focused tests pass.
